step_adapter.step_logger module

This module defines the StepLogger class, which is a wrapper around Pythons built-in logging module.

class step_adapter.step_logger.UtcFormatter(fmt=None, datefmt=None, style='%', validate=True, *, defaults=None)[source]

Bases: Formatter

This class definition creates a custom logging formatter called UtcFormatter that inherits from Pythons built-in logging.Formatter class.

Heres what the class method does: converter: This class attribute sets the time conversion function to time.gmtime, which converts a time in seconds since the epoch to a struct_time in UTC (Coordinated Universal Time) zone. This means that any timestamps generated by this formatter will be in UTC. Note that this class doesnt define any instance methods, only a class attribute.

Convert a timestamp to UTC time as struct_time.

step_adapter.step_logger.get_logger(config: LogConfig | None = None, instance_name: str | None = None) → Logger[source]

Create and configure a logger instance with the specified configuration.

This function creates a logger with customizable handlers and formatters based on the provided configuration. If no configuration is provided, it defaults to INFO level logging with a stream handler that outputs to stdout.

Parameters:

  • config (Optional[LogConfig], optional) Configuration object that specifies the logging level, handler type, format, and other logging settings. If None, defaults to INFO level with stream handler. Defaults to None.

  • instance_name (Optional[str], optional) Name identifier for the logger instance, required when using file-based handlers to create unique log filenames. Defaults to None.

Returns:

Configured logger instance ready for use.

Return type:

Logger

Raises:

ValueError If instance_name is None when using file-based handlers (LogFileHandler, LogSizeFileRotateHandler, or LogTimeFileRotateHandler).

Note

  • The function clears any existing handlers before adding the new one

  • File handlers will create the directory path if it doesnt exist

  • Rotating file handlers use custom rotator and namer functions

  • All handlers use UTF-8 encoding for file operations

  • Time-based rotation uses UTC timezone
