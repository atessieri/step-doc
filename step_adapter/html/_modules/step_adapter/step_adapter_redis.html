  1"""Redis Integration Module for STEP Adapter Framework.
  2
  3This module provides Redis-based integration components for the STEP framework,
  4enabling real-time message streaming and Crash Prediction data processing through
  5Redis Streams. It implements specialized Redis connection pooling and stream
  6management tailored for the STEP adapter architecture.
  7
  8The module contains two main classes:
  9
 10    #. **StepAdapterRedisPool**: A Redis connection pool manager that extends the base
 11       ``StepRedisPool`` with STEP-specific configurations. It provides async context
 12       manager support and optimized connection handling for the adapter workflow.
 13
 14    #. **StepAdapterRedisStream**: A Redis stream handler that manages both input
 15       message consumption and output message publishing for Crash Prediction workflows.
 16       It handles CloudEvents-compliant messaging and automatic stream lifecycle management.
 17
 18Key Features:
 19    - Async context manager support for proper resource management
 20    - CloudEvents-compliant message formatting and validation
 21    - Automatic stream TTL management based on configuration timeouts
 22    - Consumer group management for reliable message processing
 23    - Position array data publishing to Crash Prediction streams
 24    - Comprehensive logging and error handling
 25
 26Usage Example:
 27    >>> async with StepAdapterRedisPool(config, logger) as pool:
 28    ...     async with StepAdapterRedisStream(consumer_name, config, logger, pool) as stream:
 29    ...         await stream.create_client_input_stream()
 30    ...         message = await stream.receive_adapter_message()
 31    ...         await stream.publish_position_array(position_data)
 32
 33Note:
 34    This module is designed for high-throughput, real-time processing scenarios
 35    and uses memory-optimized designs with ``__slots__`` for performance.
 36"""

133class StepAdapterRedisStream(StepRedisStream):
134    """
135    This class extends StepRedisStream to provide specialized functionality for handling
136    STEP Adapter messages in a Redis streams environment. It manages both input message
137    consumption and output message publishing for Crash Prediction workflows.
138    The Adapter handles:
139
140        - Creating and managing Redis input streams with consumer groups

142        - Publishing position array data to Crash Prediction streams
143        - Automatic TTL management based on configuration timeouts
144
145    Attributes:
146        consumer_name (str): Name of the consumer for the Redis stream.
147        config (RedisStreams): Configuration object containing Redis stream settings.
148        logger (Logger): Logger instance for recording events and errors.

150    """

162        self._config = config
163        # Convert stream_ttl from milliseconds to seconds

165        super().__init__(pool=pool, logger=logger)

216        message = CrashPredictionCloudEventsJsonRoot(
