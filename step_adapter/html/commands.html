Command Line Interface

Overview

The step_adapter command line interface (CLI) provides a way to interact with the STEP Adapter.

The CLI allows users to:

  • View the program version

  • Generate JSON schema documentation for configuration

  • Create configuration templates

  • Load and run the module with a specific configuration

Basic Usage

step_adapter [options]

If no options are provided, the program will display the help message.

Options

The following command line options are available:

-h, --help

Display the help message and exit.

-v, --version

Show the program version and exit.

-s, --conf_schema <schema file>

Write the configuration JSON schema to the specified file.

-t, --conf_template <template file>

Write a configuration template to the specified file.

-c, --config <config file>

Load the configuration from the specified file and run the module.

-i, --instance <instance name>

Specify the instance name.

Note

When the module uses the -c or –config option, also the -i or –instance option must be specified to set the instance name

Detailed Command Description

Help Command

To display help information:

step_adapter --help

This will output usage instructions and a list of available options.

Version Command

To display the program version:

step_adapter --version

This will output the current version of the step_adapter.

Configuration Schema Generation

To generate a JSON schema file documenting all configuration options:

step_adapter --conf_schema /path/to/schema_file.json

This command creates a JSON schema file at the specified path. The file describes all available configuration options, their types, defaults, and constraints. The schema can be used for validation or as a reference for creating custom configurations.

Configuration Template Generation

To generate a configuration template file with default values:

step_adapter --conf_template /path/to/template_file.json

This command creates a JSON configuration file with default values for all settings. Users can modify this file as needed to customize their configuration.

Running with Configuration

To run the module with a specific configuration:

step_adapter --instance test --config /path/to/config_file.json

This loads the configuration from the specified file and starts the step_adapter. The module will connect to the Redis streams as specified in the configuration.

Configuration File Structure

The configuration file is a JSON document that defines the behavior of the step_adapter. It includes settings for:

  • Module basic settings

  • Redis client connection

  • Redis streams

  • Logging configuration

Here’s an example of the configuration structure:

{
  "templateVersion": "1",
  "pointPathLength": 10,
  "pointPathFrequency": 10,
  "periodMaxDeviationMs": 20,
  "receiveClientNumber": 3,
  "cleanerInterval": 10,
  "reconnectInterval": 2,
  "redis": {
    "connection": {
      "host": "localhost",
      "port": 6379,
      "username": "username",
      "password": "password",
      "db": 0,
      "socketTimeout": 30,
      "socketConnectTimeout": 30,
      "socketKeepIdle": 30,
      "socketKeepAlive": 30,
      "socketKeepCnt": 5,
      "sslEnable": true,
      "sslClientKeyFile": "/path/to/client.key",
      "sslClientKeyFilePassword": "password",
      "sslClientCertFile": "/path/to/client.crt",
      "sslCertRequested": true,
      "sslCaCertsFile": "/path/to/ca_certs_file.crt",
      "healthCheckInterval": 20,
      "connectionAvailableTimeout": 10,
      "protocol": 3
    },
    "streams": {
      "outputs": {
        "crashPrediction": {
          "streamId": "stream id",
          "maxLength": 1000
        }
      },
      "input": {
        "streamId": "stream id",
        "groupId": "group id",
        "waitTimeout": 30000
      }
    }
  },
  "log": {
    "handler": {
      "handlerType": "file",
      "path": ".\\logFolder",
      "filenameTrailer": ".log",
      "append": true
    },
    "level": "INFO",
    "format": "%(asctime)s.%(msecs)03d [%(levelname)s][pid=%(process)d][module=%(module)s][function=%(funcName)s]: %(message)s",
    "dateFormat": "%Y-%m-%d %H:%M:%S"
  }
}

For the STEP Adapter Configuration Schema with all configuration options, generate a schema file using the --conf_schema option.

Exit Codes

The program uses the following exit codes:

  • 0 (OK): Successful termination

  • 1 (USAGE): Command line usage error

  • 2 (ERROR): General error during execution

  • 3 (ABORTED): Operation aborted by user or system

Examples

Generate a configuration schema:

step_adapter --conf_schema ./schema.json

Create a configuration template:

step_adapter --conf_template ./template.json

Run with specific configuration:

step_adapter --instance test --config ./my_config.json

Generate both schema and template:

step_adapter --conf_schema ./schema.json --conf_template ./template.json

Program Operation

When running with a configuration file, the step_adapter performs the following operations:

  1. Connects to the Redis server using the provided connection settings

  2. Initializes an ActorDataset for managing position trajectories with temporal data management

  3. Starts multiple asynchronous tasks based on configuration:

    • Stream Message Reception: Multiple concurrent Redis stream consumer instances (based on receiveClientNumber config) continuously listen to Redis input streams for incoming CAM (Cooperative Awareness Messages), CPM (Collective Perception Messages), and VAM (Vulnerable Road User Awareness Messages)

    • Vehicle Position Processing: Processes each received CAM message by converting it to position information, inserting it into the actor dataset with cubic spline interpolation, and generating smooth position arrays

    • Position Array Publishing: When sufficient data is available, publishes interpolated position arrays to Redis output streams for crash prediction and traffic monitoring systems

    • Periodic Dataset Cleanup: Automatically removes expired trajectory data at configurable intervals (cleanerInterval) to prevent memory buildup and maintain optimal performance

  4. Automatically handles reconnection if connections are lost (using the specified reconnect interval)

  5. Manages graceful shutdown when interrupted with proper resource cleanup

The module operates as a real-time position processing pipeline, transforming raw GNSS measurements into smooth, regularly-sampled position trajectories optimized for crash prediction algorithms. It will continue running until manually interrupted (e.g., with Ctrl+C) or until a system signal is received.

Advanced Usage

Signal Handling

The step_adapter properly handles system signals for graceful shutdown. When a signal is received, it cleans up resources and terminates with the appropriate exit code.

Logging

The step_adapter includes detailed logging capabilities. Log settings can be configured in the configuration file, including:

  • Log level (DEBUG, INFO, WARNING, ERROR, CRITICAL)

  • Console logging

  • File logging with rotation options

You can enable debug logging to troubleshoot connection or message handling issues.
