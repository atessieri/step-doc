STEP Crash Prediction Schema

The STEP Crash Prediction schema defines the structure for messages incoming of STEP Crash Prediction subsystem. The schema consists of 2 main parts:

  • Header: Contains the metadata of the message like:

    • event id

    • source of the event

    • type of the event

    • time

    • data content type

    • name of the data field schema

  • Data: Contains the data of the message. This version includes:

    • Position Array v1 Schema received from STEP Adapter subsystem

This schema defines a JSON Schema version 1. For further information, please refer to the python classes.

Example of the message:

{
  "id": "56c5ecec-00ef-5a27-a5a0-bcb85a7378c1",
  "source": "adapter_1",
  "specversion": "1.0",
  "datacontenttype": "application/json",
  "time": "2018-04-05T17:31:00.000Z",
  "type": "position_array",
  "dataschema": "position_array_v1_schema",
  "data": {
    "sampleFrequency": 10,
    "vehicleLength": {
      "vehicleLengthValue": 123,
      "vehicleLengthConfidenceIndication": "noTrailerPresent"
    },
    "vehicleWidth": 16,
    "points": [
      {
        "stationInfo": {
          "stationId": 1236,
          "stationType": "passengerCar"
        },
        "objectId": 123,
        "positionType": "position_GNSS",
        "time": "2018-04-05T17:31:00.000Z",
        "position": {
          "latitude": 45.587878,
          "longitude": 60.458789,
          "positionConfidence": {
            "semiMajorAxisLengthConfidence": 12,
            "semiMinorAxisLengthConfidence": 123,
            "semiMajorAxisOrientation": 0
          },
          "altitude": {
            "value": 12000,
            "confidence": 50
          }
        },
        "speed": {
          "value": 12367,
          "confidence": 50
        },
        "heading": {
          "value": 123,
          "confidence": 50
        },
        "driveDirection": "forward"
      }
    ]
  }
}
