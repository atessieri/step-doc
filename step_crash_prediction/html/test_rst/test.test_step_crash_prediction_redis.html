test.test_step_crash_prediction_redis module

Test module for step_crash_prediction Redis integration components.

This module contains comprehensive unit tests for the Redis-based crash prediction components within the step_crash_prediction package. It focuses on testing the Redis connection pooling and stream management functionality that enables real-time data exchange between various components in the STEP (Safer Transport for Europe Platform) system.

The module provides test coverage for:

Classes Tested:

  • StepCrashPredictionRedisPool: Redis connection pool management for crash prediction services

  • StepCrashPredictionRedisStream: Redis stream operations for message publishing/consuming

Test Coverage Areas:

  • Connection pool initialization and configuration

  • Asynchronous context manager behavior

  • Redis stream creation and management

  • Message publishing to Crash Prediction streams

  • Message consumption from adapter input streams

  • CloudEvents format validation and serialization

  • Error handling and resource cleanup

Testing Methodology:

The tests extensively use mocking via pytest-mock to isolate units under test and verify proper integration with Redis infrastructure without requiring actual Redis instances. Mock objects simulate loggers, connection pools, stream configurations, and CloudEvents messages.

Note

This module requires proper configuration of Redis connection parameters and stream configurations as defined in the step_crash_prediction_config module. All tests use mocking to avoid dependencies on actual Redis instances.
