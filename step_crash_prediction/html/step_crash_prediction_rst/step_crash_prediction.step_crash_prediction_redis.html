step_crash_prediction.step_crash_prediction_redis module

Redis Integration Module for STEP Crash Prediction Framework.

This module provides Redis-based integration components for the STEP framework, enabling real-time message streaming and Crash Prediction data processing through Redis Streams. It implements specialized Redis connection pooling and stream management tailored for the STEP Crash Prediction architecture.

The module contains two main classes:

  1. StepCrashPredictionRedisPool: A Redis connection pool manager that extends the base StepRedisPool with STEP-specific configurations. It provides async context manager support and optimized connection handling for the Crash Prediction workflow.

  2. StepCrashPredictionRedisStream: A Redis stream handler that manages both input message consumption and output message publishing for Producer workflows. It handles CloudEvents-compliant messaging and automatic stream lifecycle management.

Key Features:

  • Async context manager support for proper resource management

  • CloudEvents-compliant message formatting and validation

  • Automatic stream TTL management based on configuration timeouts

  • Consumer group management for reliable message processing

  • Position array data publishing to Crash Prediction streams

  • Comprehensive logging and error handling

Usage Example:
>>> async with StepCrashPredictionRedisPool(config, logger) as pool:
...     async with StepCrashPredictionRedisStream(
            consumer_name,
            config,
            logger,
            pool
        ) as stream:
...         await stream.create_client_input_stream()
...         message = await stream.receive_crash_prediction_message()
...         await stream.publish_station_collision_array(position_data)

Note

This module is designed for high-throughput, real-time processing scenarios and uses memory-optimized designs with __slots__ for performance.

Redis connection pool Crash Prediction for the STEP framework.

This class extends StepRedisPool to provide a Redis connection pool specifically designed for use within the STEP framework. It manages Redis connections with configurable parameters and implements async context manager support for proper resource management. The pool is configured with a maximum of 2 connections by default and supports various Redis connection options including SSL configuration, socket settings, and health check intervals.

Maximum number of connections in the pool.

Note

This class uses __slots__ for memory optimization and inherits all Redis operation methods from the parent StepRedisPool class.
