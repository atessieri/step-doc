step_crash_prediction.step_logger module

Logging utilities for the step_crash_prediction package.

This module provides comprehensive logging functionality with customizable handlers, UTC timezone formatting, and automatic log file compression. It supports various logging configurations including console output, file logging, and rotating file handlers with size or time-based rotation.

Key Features:

  • UTC timezone formatting for consistent timestamps across different environments

  • Automatic log file compression using BZIP2 when files are rotated

  • Support for multiple handler types: stream, null, file, and rotating file handlers

  • Configurable logging levels and formats

  • Thread-safe logging operations

  • Automatic directory creation for file-based logging

Example

>>> from step_crash_prediction.step_logger import get_logger
... from step_crash_prediction.step_crash_prediction_config import LogConfig, LogFileHandler
...
... config = LogConfig(
...     level=LogMode.DEBUG,
...     handler=LogFileHandler(path="/var/log/app", filenameTrailer=".log")
... )
... logger = get_logger(config=config, instance_name="my_app")
... logger.debug("Debug message")

Note

All file operations use UTF-8 encoding, and rotated log files are automatically compressed using BZIP2 compression with maximum compression level for optimal storage efficiency.

This class definition creates a custom logging formatter called UtcFormatter that inherits from Python’s built-in logging.Formatter class.

Here’s what the class method does: converter: This class attribute sets the time conversion function to time.gmtime, which converts a time in seconds since the epoch to a struct_time in UTC (Coordinated Universal Time) zone. This means that any timestamps generated by this formatter will be in UTC. Note that this class doesn’t define any instance methods, only a class attribute.

step_crash_prediction.step_logger.get_logger(config: LogConfig | None = None, instance_name: str | None = None) → Logger[source]

Create and configure a logger instance with the specified configuration.

This function creates a logger with customizable handlers and formatters based on the provided configuration. If no configuration is provided, it defaults to INFO level logging with a stream handler that outputs to stdout.

Parameters:

  • config (Optional[LogConfig], optional) – Configuration object that specifies the logging level, handler type, format, and other logging settings. If None, defaults to INFO level with stream handler. Defaults to None.

  • instance_name (Optional[str], optional) – Name identifier for the logger instance, required when using file-based handlers to create unique log filenames. Defaults to None.

Returns:

Configured logger instance ready for use.

Return type:

Logger

Raises:

ValueError – If instance_name is None when using file-based handlers (LogFileHandler, LogSizeFileRotateHandler, or LogTimeFileRotateHandler).

Note

  • The function clears any existing handlers before adding the new one

  • File handlers will create the directory path if it doesn’t exist

  • Rotating file handlers use custom rotator and namer functions

  • All handlers use UTF-8 encoding for file operations

  • Time-based rotation uses UTC timezone
