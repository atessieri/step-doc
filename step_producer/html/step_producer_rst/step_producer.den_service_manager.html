step_producer.den_service_manager module

DENM Service Manager Module.

This module provides the core management functionality for DENM (Decentralized Environmental Notification Message) services in an ITS (Intelligent Transportation Systems) environment.

Overview:

The module implements a comprehensive DENM lifecycle management system that handles the creation, processing, termination, and cleanup of DENM messages. It serves as the primary interface for managing collision-related environmental notifications between ITS stations.

Key Components:

  • DenServiceManager: Main service class that orchestrates DENM operations

  • Station Collision Processing: Handles incoming collision data and generates DENM messages

  • Message Termination: Processes DENM negations and cancellations

  • Periodic Cleanup: Automatically manages expired DENM records

  • Redis Stream Integration: Publishes messages to Redis streams for distribution

Core Functionality:

  1. Collision Event Processing: Converts station collision arrays into DENM messages

  2. Message Lifecycle Management: Tracks active DENM records from creation to expiration

  3. Termination Handling: Processes incoming negation messages and removes corresponding records

  4. Automatic Cleanup: Periodically identifies and cancels expired DENM messages

  5. Stream Publishing: Publishes DENM messages and cancellations to Redis streams

Architecture:

The module follows an asynchronous architecture pattern, utilizing:

  • Async/await for non-blocking operations

  • Redis streams for message distribution

  • Data managers for state persistence

  • Configurable timing parameters for adaptive behavior

Dependencies:

  • step_cloud_schemas: DENM data structures and message schemas

  • step_producer: Redis connection pooling and stream management

  • asyncio: Asynchronous operation support

  • logging: Comprehensive operation logging

Usage Pattern:

The module is typically instantiated with station-specific configuration and then used to process incoming collision data while automatically managing the lifecycle of generated DENM messages through periodic cleanup operations.

Example

>>> manager = DenServiceManager(
...     station_id=12345,
...     time_after_collision=5000,
...     denm_last_update_max_delay_ms=1000,
...     denm_check_last_update_max_interval=500,
...     logger=logger
... )
...
... # Process collision events
... await manager.process_station_collisions_received(collision_data, stream)
...
... # Handle incoming DENM records
... await manager.process_denm_record_received(denm_message)
...
... # Start automatic cleanup
... await manager.periodical_cleanup_run(redis_pool, consumer_name, streams)
Thread Safety:

The module is designed for single-threaded async operation. Multiple instances can be used across different async contexts, but individual instances should not be shared between concurrent tasks without proper synchronization.

Performance Considerations:

  • Utilizes slots for memory efficiency

  • Implements periodic cleanup to prevent memory leaks

  • Configurable intervals allow tuning for different traffic scenarios

  • Redis streams provide scalable message distribution

Manages DENM (Decentralized Environmental Notification Message) services for ITS (Intelligent Transportation Systems). The DenServiceManager class handles the lifecycle of DENM messages, including processing incoming station collision data, managing DENM records, handling message terminations, and performing periodic cleanup of expired records. Key responsibilities:

  • Processing station collision arrays and generating corresponding DENM messages

  • Handling incoming DENM records and processing termination requests

  • Creating cancellation messages for DENM events

  • Performing periodic cleanup of expired DENM records

  • Publishing DENM messages to Redis streams

async periodical_cleanup_run(redis_pool: StepProducerRedisPool, instance_name: str, streams: RedisStreams) → None[source]

Periodically performs cleanup of expired DENM records and publishes cancellation messages for them.

This asynchronous method runs in an infinite loop, checking for expired DENM records, publishing cancellation messages for each expired record, and then sleeping for a calculated interval before repeating the process.

Parameters:

  • redis_pool (StepProducerRedisPool) – The Redis connection pool to use for stream operations.

  • instance_name (str) – The name of the instance for the Redis stream.

  • streams (RedisStreams) – Configuration for the Redis streams.

Returns:

None

Note

The method logs each step of the cleanup process, including when it runs, which records are expired, and when it sleeps before the next run.
