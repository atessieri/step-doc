test.test_den_service_manager module

Test module for DenServiceManager class functionality.

This module contains comprehensive unit tests for the DenServiceManager class, which is responsible for managing Decentralized Environmental Notification Messages (DENM) in the STEP Producer system. The DenServiceManager handles collision detection, message processing, termination handling, and periodic cleanup of expired DENM records.

Test Coverage:

  • Initialization and dependency injection verification

  • Station collision array processing and DENM record publishing

  • DENM record termination handling (NEGATION, CANCELLATION, None)

  • Cancellation message creation with proper field mapping

  • Periodic cleanup operations for expired DENM records

  • Redis stream integration for message publishing

  • Error handling and edge cases (empty collision lists)

Key Components Tested:

  • DenServiceManager initialization with proper parameter passing

  • Integration with DenServiceDataManager for data operations

  • Redis stream publishing for ITS messages

  • DENM record lifecycle management (creation, update, termination)

  • Asynchronous cleanup processes with proper interval calculations

The test suite uses extensive mocking to isolate the DenServiceManager behavior from external dependencies, ensuring reliable and fast unit tests that verify business logic without requiring actual Redis connections or external services.

Test suite for the DenServiceManager class. This class contains unit and integration tests for the DenServiceManager, verifying its initialization, DENM record processing, station collision handling, cancellation message creation, and periodic cleanup logic. The tests utilize pytest and pytest-mock to mock dependencies and assert correct behavior in various scenarios, including:

  • Initialization and attribute assignment.

  • Processing of station collision arrays and publishing of DENM records.

  • Handling of empty collision lists.

  • Processing of DENM records with different termination types (NEGATION, CANCELLATION, None).

  • Creation of cancellation DENM messages with expected field values.

  • Periodic cleanup and publishing of cancellation messages for expired records.

Mocks are used for external dependencies such as loggers, Redis streams, and service data managers to ensure isolated and reliable testing of DenServiceManager logic.
