  1"""DENM Service Manager Module.
  2
  3This module provides the core management functionality for DENM (Decentralized Environmental
  4Notification Message) services in an ITS (Intelligent Transportation Systems) environment.
  5
  6Overview:
  7    The module implements a comprehensive DENM lifecycle management system that handles the
  8    creation, processing, termination, and cleanup of DENM messages. It serves as the primary
  9    interface for managing collision-related environmental notifications between ITS stations.
 10
 11Key Components:
 12
 13    - **DenServiceManager**: Main service class that orchestrates DENM operations
 14    - **Station Collision Processing**: Handles incoming collision data and generates DENM messages
 15    - **Message Termination**: Processes DENM negations and cancellations
 16    - **Periodic Cleanup**: Automatically manages expired DENM records
 17    - **Redis Stream Integration**: Publishes messages to Redis streams for distribution
 18
 19Core Functionality:
 20
 21    #. **Collision Event Processing**: Converts station collision arrays into DENM messages
 22    #. **Message Lifecycle Management**: Tracks active DENM records from creation to expiration
 23    #. **Termination Handling**: Processes incoming negation messages and removes corresponding
 24       records
 25    #. **Automatic Cleanup**: Periodically identifies and cancels expired DENM messages
 26    #. **Stream Publishing**: Publishes DENM messages and cancellations to Redis streams
 27
 28Architecture:
 29    The module follows an asynchronous architecture pattern, utilizing:
 30
 31        - Async/await for non-blocking operations
 32        - Redis streams for message distribution
 33        - Data managers for state persistence
 34        - Configurable timing parameters for adaptive behavior
 35
 36Dependencies:
 37
 38    - step_cloud_schemas: DENM data structures and message schemas
 39    - step_producer: Redis connection pooling and stream management
 40    - asyncio: Asynchronous operation support
 41    - logging: Comprehensive operation logging
 42
 43Usage Pattern:
 44    The module is typically instantiated with station-specific configuration and then used
 45    to process incoming collision data while automatically managing the lifecycle of
 46    generated DENM messages through periodic cleanup operations.
 47
 48Example:
 49    >>> manager = DenServiceManager(
 50    ...     station_id=12345,
 51    ...     time_after_collision=5000,
 52    ...     denm_last_update_max_delay_ms=1000,
 53    ...     denm_check_last_update_max_interval=500,
 54    ...     logger=logger
 55    ... )
 56    ...
 57    ... # Process collision events
 58    ... await manager.process_station_collisions_received(collision_data, stream)
 59    ...
 60    ... # Handle incoming DENM records
 61    ... await manager.process_denm_record_received(denm_message)
 62    ...
 63    ... # Start automatic cleanup
 64    ... await manager.periodical_cleanup_run(redis_pool, consumer_name, streams)
 65
 66Thread Safety:
 67    The module is designed for single-threaded async operation. Multiple instances can
 68    be used across different async contexts, but individual instances should not be
 69    shared between concurrent tasks without proper synchronization.
 70
 71Performance Considerations:
 72
 73    - Utilizes slots for memory efficiency
 74    - Implements periodic cleanup to prevent memory leaks
 75    - Configurable intervals allow tuning for different traffic scenarios
 76    - Redis streams provide scalable message distribution
 77"""

102    Manages DENM (Decentralized Environmental Notification Message) services
103    for ITS (Intelligent Transportation Systems).
104    The DenServiceManager class handles the lifecycle of DENM messages, including processing

107    Key responsibilities:
108
109        - Processing station collision arrays and generating corresponding DENM messages

113        - Publishing DENM messages to Redis streams

116            station_id: Unique identifier for the station
117            time_after_collision: Time duration after collision event in appropriate units
118            denm_last_update_max_delay_ms: Maximum delay in milliseconds for DENM last update
119            denm_check_last_update_max_interval: Maximum interval for checking DENM last update
120            logger: Logger instance for logging operations

123        >>> manager = DenServiceManager(
124        ...    station_id=12345,
125        ...    time_after_collision=5000,
126        ...    denm_last_update_max_delay_ms=1000,

171    async def process_station_collisions_received(
172        self,
173        station_collision_array: StationCollisionArrayV1,
174        stream: StepProducerRedisStream,
175    ) -> None:
176        """Processes a received array of station collisions, inserts them into the DENM
177        service data, and publishes each resulting DENM record to the provided Redis
178        stream.
179
180        Executes the following steps:
181
182            - Inserts station collision data into the DENM service.
183            - Publishes DENM messages to the specified Redis stream.
184            - Logs each DENM record being published.
185
186        Args:
187            station_collision_array (StationCollisionArrayV1): The array
188                of station collision data to process.
189            stream (StepProducerRedisStream): The Redis stream to which
190                DENM messages will be published.
191
192        Returns:
193            None
194        """
195        denm_list = self._denm_service_data.insert_station_collision_array(
196            station_collision_array=station_collision_array
197        )
198        for denm_record in denm_list:
199            self._logger.debug(msg=f"Publishing DENM record: {denm_record}")
200            await stream.publish_its_message(
201                message=denm_record.denm,
202            )

204    async def process_denm_record_received(self, denm: DenmDataV1) -> None:
205        """Processes a received DENM (Decentralized Environmental Notification Message)
206        record.
207
208        If the DENM record's management container indicates a termination with a negation,
209        logs the receipt of the negation and removes the corresponding DENM record from
210        the service data.
211
212        Args:
213            denm (DenmDataV1): The DENM data object containing management information.
214
215        Returns:
216            None
217        """
218        if (denm.managementContainer.termination is not None) and (
219            denm.managementContainer.termination == Termination.NEGATION
220        ):
221            self._logger.info(
222                msg=f"Received DENM negation for actionId: {denm.managementContainer.actionId}"
223            )
224            self._denm_service_data.remove_denm_record(
225                denm_id=denm.managementContainer.actionId
226            )

228    def _denm_create_cancellation(self, denm: DenmDataV1) -> DenmDataV1:

230        event.
231

234        All other fields are preserved from the original DENM message.
235

238
239        Returns:

241        """
242        return DenmDataV1(
243            stationId=denm.stationId,
244            managementContainer=ManagementContainer(
245                actionId=denm.managementContainer.actionId,
246                detectionTime=denm.managementContainer.detectionTime,
247                referenceTime=denm.managementContainer.referenceTime,
248                termination=Termination.CANCELLATION,
249                eventPosition=denm.managementContainer.eventPosition,
250                awarenessDistance=denm.managementContainer.awarenessDistance,
251                trafficDirection=denm.managementContainer.trafficDirection,
252                validityDuration=denm.managementContainer.validityDuration,
253                transmissionInterval=denm.managementContainer.transmissionInterval,
254                stationType=denm.managementContainer.stationType,
255            ),
256            situationContainer=denm.situationContainer,

261    async def periodical_cleanup_run(
262        self,
263        redis_pool: StepProducerRedisPool,
264        instance_name: str,
265        streams: RedisStreams,
266    ) -> None:
267        """Periodically performs cleanup of expired DENM records and publishes
268        cancellation messages for them.
269
270        This asynchronous method runs in an infinite loop, checking for expired DENM records,
271        publishing cancellation messages for each expired record, and then sleeping
272        for a calculated interval before repeating the process.
273
274        Args:
275            redis_pool (StepProducerRedisPool): The Redis connection pool to use for
276                stream operations.
277            instance_name (str): The name of the instance for the Redis stream.
278            streams (RedisStreams): Configuration for the Redis streams.
279
280        Returns:
281            None
282
283        Notes:
284            The method logs each step of the cleanup process, including when it runs,
285            which records are expired, and when it sleeps before the next run.
286        """
287        async with StepProducerRedisStream(
288            consumer_name=instance_name,
289            config=streams,
290            logger=self._logger,
291            pool=redis_pool,
292        ) as redis_stream:
293            while True:
294                self._logger.debug(msg="Running DENM periodical cleanup")
295                (expired_records, oldest_record) = (
296                    self._denm_service_data.get_denm_records_clean_reference_expired()
297                )
298                for record in expired_records:
299                    self._logger.debug(
300                        msg=f"Publishing DENM cancellation: key {record.record_key}, "
301                        f"reference time {record.reference_time}, "
302                        f"validity duration {record.denm.managementContainer.validityDuration}"
303                    )
304                    await redis_stream.publish_its_message(
305                        message=self._denm_create_cancellation(denm=record.denm),
306                    )
307                sleep_time = (
308                    self._denm_service_data.get_interval_oldest_update_ms(
309                        oldest_record=oldest_record
310                    )
311                    / 1000
312                )
313                self._logger.debug(
314                    msg=f"Sleeping for {sleep_time:.3f} seconds before next cleanup run"
315                )
316                await asyncio_sleep(delay=sleep_time)
