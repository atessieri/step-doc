  1"""STEP Gateway Module for MQTT and Redis Integration.
  2
  3This module provides the necessary components for a secure, configurable MQTT-based gateway
  4within the STEP client ecosystem. It forms a crucial bridge between MQTT messaging systems
  5and Redis streams, enabling interoperability between different ITS (Intelligent Transport Systems)
  6components.
  7
  8The module implements a gateway that supports:
  9
 10    - Secure TLS/SSL MQTT communications with configurable parameters
 11    - Bidirectional message conversion between MQTT and Redis streams
 12    - Asynchronous operation with context management for clean resource handling
 13    - Quality of Service (QoS) controls for reliable message delivery
 14    - Support for various ITS message types, with focus on CAM (Cooperative Awareness Messages)
 15    - Clean error handling and comprehensive logging
 16
 17The StepGw class leverages async/await patterns throughout to ensure non-blocking
 18operations, making it suitable for high-throughput messaging environments in
 19ITS applications.
 20"""

 47    """
 48    StepGw - Step Gateway Client implementing AsyncComManager
 49
 50    This class provides a high-level client for connecting to a STEP service
 51    using MQTT protocol. It handles secure connections, topic subscriptions
 52    and message management according to the provided configuration.
 53
 54    The class manages:
 55
 56        - TLS/SSL connections with certificate validation
 57        - MQTT connection parameters (host, port, auth, timeouts, etc.)
 58        - Topic subscriptions with configurable QoS levels
 59        - Message queuing and delivery
 60
 61    It implements the AsyncComManager interface and supports asynchronous
 62    context manager protocol for safe resource management.
 63
 64    Attributes:
 65        config (StepConfig): Configuration object containing STEP client settings.
 66        logger (Logger): Logger instance for logging messages and errors.
 67
 68    Example:
 69        >>> config = StepConfig(...)
 70        ... logger = Logger(...)
 71        ... async with StepGw(config, logger) as step_client:
 72        ...    # Use the STEP client
 73    """

127            max_inflight_messages=self._config.stepClient.mqttProtocol.maxInflightMessages,

174    async def subscribe_execution(self) -> None:
175        """Subscribe to a list of topics with specified MQTT options.
176
177        This method subscribes to a list of topics specified in the configuration
178        with the following MQTT options:
179
180            - QoS level from config
181            - No local messages (noLocal=True)
182            - Do not retain as published (retainAsPublished=False)
183            - Do not send retained messages (retainHandling=RETAIN_DO_NOT_SEND)
184
185        Raises:
186            StepClientException: If subscription fails or QoS level doesn't match the configured
187                one.
188
189        Returns:
190            None
191        """
192        if self._config.stepClient.mqttProtocol.subscriptionTopics is not None:
193            sub_results = await self.subscribe(
194                topic=[
195                    (
196                        topic,
197                        SubscribeOptions(
198                            qos=self._config.stepClient.mqttProtocol.maxSubscriptionQos,
199                            noLocal=True,
200                            retainAsPublished=False,
201                            retainHandling=SubscribeOptions.RETAIN_DO_NOT_SEND,
202                        ),
203                    )
204                    for topic in self._config.stepClient.mqttProtocol.subscriptionTopics
205                ],
206            )
207            for topic, result in zip(
208                self._config.stepClient.mqttProtocol.subscriptionTopics,
209                sub_results,
210            ):
211                if result != self._config.stepClient.mqttProtocol.maxSubscriptionQos:
212                    raise StepClientException(
213                        f"Failed to subscribe to {topic}: {result}"
214                    )
215                self._logger.info(msg=f"Subscribed to {topic}: {result}")
