test.test_step_broker module

Unit tests for the StepBroker MQTT gateway component.

This module contains comprehensive unit tests for the StepBroker class, which serves as a critical component in the STEP (Safer Transport for Europe Platform) client system for Intelligent Transport Systems (ITS). The StepBroker acts as an MQTT-based gateway that facilitates secure communication between ITS applications and messaging infrastructure.

Test Coverage:

The test suite provides thorough coverage of the following StepBroker functionality:

Configuration & Initialization:

  • SSL/TLS enabled and disabled MQTT configurations

  • Proper initialization of GeoNetworkAddress with traffic participant types

  • Configuration of TLS parameters with certificates and security settings

  • AsyncComManagerParams setup with comprehensive MQTT settings

Asynchronous Context Management:

  • Context manager entry and exit behavior

  • Resource management and cleanup verification

MQTT Subscription Management:

  • Successful topic subscriptions with QoS level validation

  • Error handling for QoS level mismatches

  • Handling of empty, None, or malformed subscription configurations

  • Validation of subscription result counts against configured topics

ITS Message Publishing:

  • Publishing of FacilityMessage objects through MQTT

  • GPS location integration with ITS timestamps

  • Geo-network address management for mobile and stationary stations

Error Handling & Exception Management:

  • StepClientException propagation for configuration errors

  • Comprehensive error scenarios and edge cases

  • Logging verification for debugging and monitoring

Testing Approach:

All tests employ extensive mocking to isolate the unit under test and eliminate external dependencies including:

  • MQTT broker connections

  • SSL certificate validation

  • Network I/O operations

  • Redis connections

  • File system access

The test suite uses pytest with async support and MockerFixture for creating clean, isolated test environments that can run independently without requiring actual MQTT infrastructure or ITS hardware.

Dependencies:

  • pytest: Test framework with async support

  • pytest-mock: Mock object creation and patching

  • aiomqtt: MQTT client library for TLS parameter testing

  • step_cloud_schemas: ITS message schema definitions

  • step_lib: Core STEP library components

  • step_client: The module under test

Note

This test module is specifically designed for the STEP client ecosystem and follows ITS standards for message exchange and communication protocols. The tests ensure compliance with MQTT QoS requirements and secure TLS communication standards expected in transportation infrastructure.

Test suite for the StepBroker class.

This test class contains comprehensive unit tests for the StepBroker class, which is responsible for managing MQTT communication and ITS message handling in the STEP client system. The tests cover initialization, SSL configuration, subscription management, and message publishing functionality. The test suite includes the following test scenarios:

  • Initialization with SSL enabled and disabled configurations

  • Asynchronous context manager entry behavior

  • Subscription execution with various scenarios:

    • Successful subscription with matching QoS levels

    • Exception handling for QoS level mismatches

    • Handling of empty or None subscription topics

    • Error handling for subscription result count mismatches

  • ITS message publishing functionality

All tests use mocking to isolate the unit under test and avoid external dependencies such as actual MQTT brokers, SSL certificates, and network connections.
