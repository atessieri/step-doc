step_client.step_client_redis module

STEP Client Redis Integration.

This module provides Redis-based communication functionality for the STEP client, enabling interaction with the STEP ecosystem through Redis streams. It implements connection pooling and stream handling specifically tailored for Intelligent Transport Systems (ITS) message exchange between clients, adapters, and producers.

The module contains two main classes:

  1. StepClientRedisPool: Extends StepRedisPool to provide a Redis connection pool specifically configured for STEP client needs, handling connection parameters and management.

  2. StepClientRedisStream: Extends StepRedisStream to implement stream handling for ITS messages (CAM, DENM) according to the CloudEvents specification, including:

    • Creating and managing input streams with group IDs

    • Publishing messages to adapter and producer streams

    • Receiving and parsing messages from configured streams

    • Converting between CloudEvents format and STEP facility messages

This module serves as the communication layer for STEP clients, enabling standardized message exchange in ITS applications through Redis infrastructure.

class step_client.step_client_redis.StepClientRedisStream(consumer_name: str, config: RedisStreams, logger: Logger, pool: StepRedisPool)[source]

Bases: StepRedisStream

Redis stream implementation for the STEP client. This class extends the StepRedisStream to provide specific functionality for handling ITS (Intelligent Transport Systems) messages in a Redis stream context. It manages the creation of input streams and handles publishing and receiving of different types of ITS messages (CAM, DENM) with appropriate formatting.

The class implements:

  • Input stream creation with configured group IDs

  • Publishing of ITS messages to adapter and producer streams

  • Receiving and parsing of ITS messages from configured streams

Each message is formatted according to CloudEvents specifications and handled based on its message type.

Publishes ITS messages to appropriate Redis streams based on message type.

This method handles different types of facility messages (CAM, DENM) and publishes them to configured Redis streams in the appropriate format.

For CAM messages:

  • Publishes to the adapter stream only

For DENM messages:

  • Publishes to both adapter and producer streams
