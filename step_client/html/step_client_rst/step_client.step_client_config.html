step_client.step_client_config module

STEP Client Configuration Module.

This module defines the configuration system for the STEP (Standardized Transport and Environment Protocol) client. It provides a comprehensive set of Pydantic models that describe the configuration structure with strong typing, validation rules, and schema generation capabilities.

The module is organized in a hierarchical structure, where the top-level StepClientConfig class contains nested configuration components for different aspects of the system:

  • MQTT Client configuration (connection parameters, security settings)

  • MQTT Protocol configuration (QoS, topic management, session parameters)

  • Redis Client configuration (for message broker integration)

  • Logging configuration (handlers, levels, formatting)

Each configuration class provides:

  • Type validation and constraints via Pydantic fields

  • Detailed documentation

  • JSON Schema generation for configuration file validation

  • Default values through get_default() static methods

The configuration system is designed to be immutable, with strict validation to prevent configuration errors at runtime. Each component features comprehensive documentation to assist users in understanding the available options and their effects.
