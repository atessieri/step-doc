Package Script

This section describes how to use the script inside the project. The script is different for Windows and Linux.

Note

The scripts use uv program. If it is not installed, it is necessary to install it first using pip install uv

Create environment

The script creates a virtual environment in the project folder. The virtual environment is created in the folder .venv and the script will install all the dependencies in this folder.

$.\create_env.bat <python version>

$source create_env.sh <python version>

After the environment is created, it is necessary to activate it. The activation is done with the command:

$.venv\Script\activate

$source .venv\bin\activate

Make

The script can be used on windows or linux with make installed. The commands available are:

  1. Clean all temporary folder like build, schemas, rts_generated, etc:

    $.\make.bat clean

    $make clean

  2. Execute the tests of the package:

    $.\make.bat test

    $make test

  3. Check if all links in the documentation are valid:

    $.\make.bat checklink

    $make checklink

  4. Create the documentation in html format:

    $.\make.bat html

    $make html

  5. Create the documentation in latex format:

    $.\make.bat latex

    $make latex

  6. Upload the package to the registry:

    $.\make.bat upload

    $make upload

  7. List of all commands available:

    $.\make.bat help

    $make help

PDF conversion

In order to create the pdf documentation, following the steps:

  1. Change the folder to the build folder:

    $cd build/latex

  2. Create the documentation in pdf format:

    $build/latex/make
