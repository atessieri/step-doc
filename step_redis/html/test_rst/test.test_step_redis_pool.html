test.test_step_redis_pool module

Unit tests for step_redis_pool module.

This module contains unit tests for the StepRedisPool and StepRedisPoolParams classes from the step_redis.step_redis_pool module. These tests verify the correct behavior of Redis connection pool creation, configuration, and life cycle management.

Note

These tests use mocking to avoid requiring a real Redis server.

class test.test_step_redis_pool.TestStepRedisPool[source]

Bases: object

Test cases for the StepRedisPool class. This class contains tests to verify the proper functionality of StepRedisPool, which extends Redis’s BlockingConnectionPool to provide additional features like SSL support and async context manager capabilities. The test suite covers:

  1. Initialization with SSL configuration

  2. Initialization without SSL configuration

  3. Async context manager behavior

Each test case uses pytest fixtures to provide mocked dependencies and test-specific configuration.

class test.test_step_redis_pool.TestStepRedisPoolParams[source]

Bases: object

TestStepRedisPoolParams is a test class for verifying the correct initialization and configuration of StepRedisPoolParams objects. This test class ensures that the StepRedisPoolParams class properly handles both SSL-enabled and non-SSL connection configurations, including all relevant connection parameters like host, port, authentication credentials, timeouts, socket settings, and SSL certificate options.

The class contains tests to verify:

  • SSL configuration with certificates, keys, and related settings

  • Non-SSL configuration with appropriate default values

  • Proper initialization of all connection parameters in both scenarios

test.test_step_redis_pool.redis_params_ssl_function() → StepRedisPoolParams[source]

Creates and returns a StepRedisPoolParams object with SSL configuration.

This function defines a set of parameters for connecting to a Redis server with SSL/TLS encryption enabled, including authentication credentials, connection timeouts, keep-alive settings, and SSL certificate configuration.

Returns:

A configuration object with the following settings:

  • host: Redis server hostname (redis.example.com)

  • port: Redis server port (6379)

  • username/password: Authentication credentials

  • db: Redis database index (0)

  • socket_timeout: Socket operation timeout in seconds (10)

  • socket_connect_timeout: Connection timeout in seconds (5)

  • socket_keep_* parameters: TCP keep-alive settings

  • SSL parameters: Enables SSL with client certificate authentication

  • health_check_interval: Pool health check interval in seconds (30)

  • max_connections: Maximum connections in the pool (10)

  • connection_available_timeout: Timeout for getting a connection from pool (5)

  • protocol: Redis protocol version (2)
