step_lib package

STEP Library Package.

A comprehensive library for handling STEP (Simulation of Transportation Environment Platform) communications and message formatting. This package provides tools and utilities for:

  • Asynchronous and synchronous communication management

  • ITS (Intelligent Transportation Systems) message handling

  • Message supported:

    • CAM: Cooperative Awareness Message

    • DENM: Decentralized Environmental Notification Message

    • PCM: Collective Perception Message

    • VAM: VRU Awareness Message

  • MQTT-based communication with configurable QoS

  • Abstract message handling facilities

Main components:

  • AsyncComManager: Handles asynchronous MQTT communications

  • SyncComManager: Manages synchronous communications

  • CamMessageV1/DenmMessageV1: ITS message implementations

  • FacilitySerializer: ITS-specific functionality

This class extends the aiomqtt.Client class and provides additional functionality for publishing and subscribing to MQTT topics to Step broker. The initialization process:

  1. Sets up logging

  3. Initializes base MQTT client with provided parameters

  4. Setup message serialization for ITS messages

Abstract base class representing a facility message in the system.

This class defines the interface for all facility messages, providing common functionality and required methods that subclasses must implement. A facility message represents a standardized communication unit that can be serialized, transmitted, and deserialized between different system components. FacilityMessage objects contain metadata about the message (time, source, location) and provide methods for conversion between different formats (dictionary, JSON, ASN.1). All subclasses must implement the abstract methods to provide specific behavior for each message type. The class supports equality comparison based on timestamp.

Bases: object

GeoNetworkAddress class representing a Geo-Networking address according to ETSI TS 103 836-4-1 specification. This class implements the GeoNetworking address format which includes:

  • A manual configuration flag (M)

  • Traffic participant type (TPT)

  • MAC ID (48-bit identifier)

The GeoNetworkAddress is an 8-byte structure where:

  • First 2 bytes contain the M flag (1 bit), TPT (5 bits), and reserved bits (10 bits)

  • Last 6 bytes contain the MAC ID in network byte order (big endian)

class step_lib.MqttQoS(*values)[source]

Bases: IntEnum

MqttQoS: Enumeration class for MQTT Quality of Service levels.

The MQTT protocol provides three QoS levels for message delivery:

QOS_0

At most once delivery. The message is delivered according to the best efforts of the operating environment. Message loss can occur.

QOS_1

At least once delivery. The message is assured to arrive but duplicates may occur.

QOS_2

Exactly once delivery. The message is assured to arrive exactly once. This is the highest quality of service level and has the highest overhead.

Synchronous Communication Manager for MQTT messaging. This class provides a thread-safe interface for MQTT communication, managing operations through queues and background threads.

The manager handles:

  • MQTT message publishing and subscribing

  • Automatic reconnection on connection loss

  • Thread-safe message queuing

  • Exception handling and logging

params

Configuration parameters for the communication manager, containing all settings (e.g. maximum queue sizes).

Type:

SyncComManagerParams

The class can be used either with context manager (recommended) or manual start/stop:

Example

>>> with SyncComManager(params) as manager:
        manager.subscribe(['topic1', 'topic2'], QoS.AT_LEAST_ONCE)
        manager.publish_its_message(message=message, is_station_mobile=True)

Or manually:

Example

>>> manager = SyncComManager(params)
>>> manager.start()
>>> manager.subscribe(['topic1'], QoS.AT_MOST_ONCE)
>>> manager.stop()
