step_lib.sync_com_manager module

Synchronous MQTT Communication Manager Module.

This module provides a thread-safe interface for MQTT communication with automatic reconnection and message queueing capabilities. It wraps asynchronous MQTT operations in a synchronous interface making it easier to use in traditional threaded applications.

Key Features:

  • Thread-safe MQTT operations

  • Automatic reconnection handling

  • Message queueing for incoming and outgoing messages

  • Support for QoS levels

  • Configurable connection parameters

  • Context manager support

  • Proper cleanup of resources

Example

>>> params = SyncComManagerParams.get_default()
>>> with SyncComManager(params) as manager:
>>>     manager.subscribe(['topic1'], MqttQoS.AT_LEAST_ONCE)
>>>     manager.publish_its_message(message=some_message, is_station_mobile=True)
>>>     received = manager.get_its_message_received()

class step_lib.sync_com_manager.SyncComManager(params: SyncComManagerParams)[source]

Bases: object

Synchronous Communication Manager for MQTT messaging. This class provides a thread-safe interface for MQTT communication, managing operations through queues and background threads.

The manager handles:

  • MQTT message publishing and subscribing

  • Automatic reconnection on connection loss

  • Thread-safe message queuing

  • Exception handling and logging

params

Configuration parameters for the communication manager, containing all settings (e.g. maximum queue sizes).

Type:

SyncComManagerParams

The class can be used either with context manager (recommended) or manual start/stop:

Example

>>> with SyncComManager(params) as manager:
        manager.subscribe(['topic1', 'topic2'], QoS.AT_LEAST_ONCE)
        manager.publish_its_message(message=message, is_station_mobile=True)

Or manually:

Example

>>> manager = SyncComManager(params)
>>> manager.start()
>>> manager.subscribe(['topic1'], QoS.AT_MOST_ONCE)
>>> manager.stop()

Parameters class for synchronous communication manager configuration. This class extends AsyncComManagerParams to provide configuration parameters specifically for synchronous communication management. It includes settings for reconnection intervals and queue management.

reconnect_interval

Time interval in seconds between reconnection attempts when the connection is lost. Defaults to 2 seconds.

Type:

int

max_incoming_queue_rooms

Maximum number of slots available in the incoming message queue. Defaults to 60 slots.

int
