step_lib.geonet.btp_message module

Basic Transport Protocol (BTP) Message implementation for GeoNetworking.

This module provides classes for handling Basic Transport Protocol (BTP) messages as defined in ETSI TS 103 836-5-1. BTP is a transport layer protocol designed for Intelligent Transport Systems (ITS) that operates on top of the GeoNetworking protocol.

The module includes:

  • BtpMessageType: An enumeration of BTP message types (BTP-A and BTP-B)

  • BtpMessage: A dataclass representing a BTP message with serialization/deserialization capabilities

BTP provides end-to-end, connection-less transport service in the ITS network stack:

  • BTP-A handles interactive packet transport with source and destination ports

  • BTP-B handles non-interactive packet transport with destination port and port info

These implementations can be used to create, parse, and manipulate BTP messages within a GeoNetworking communication stack for vehicular networking applications.

Bases: object

Basic Transport Protocol (BTP) Message. This class represents a BTP message as defined in the ETSI TS 103 836-5-1 V2.1.1 standard. BTP provides an end-to-end connection-less transport service for GeoNetworking data. There are two variants of BTP:

  • BTP-A (Interactive): Used for interactive communication like CAM and DENM messages

  • BTP-B (Non-interactive): Used for non-interactive communication like one-way broadcasts

Source port number (0-65535) for BTP-A messages, None for BTP-B messages

Type:

Type:

int

destination_port_info

Additional destination port information (0-65535) for BTP-B messages, None for BTP-A messages

Type:

Optional[int]

Raises:

ValueError – If any port number is outside the valid range of 0-65535 (2-byte value) BTP messages are part of the ITS station architecture and are typically carried as payload in GeoNetworking packets.

__post_init__() → None[source]

Validates the ports after initialization of the instance.

This method performs validation checks on the port values:

  • source_port (if provided): must be between 0 and 65535 (2 bytes)

  • destination_port: must be between 0 and 65535 (2 bytes)

  • destination_port_info (if provided): must be between 0 and 65535 (2 bytes)

Raises:

ValueError – If any of the port values are outside the valid range of 0-65535.

Note

The validation logic is optimized by first checking for any potential issues with a combined condition before performing individual checks to determine which specific port value failed validation.

class step_lib.geonet.btp_message.BtpMessageType(*values)[source]

Bases: StrEnum

Enumeration class for BTP (Basic Transport Protocol) message types.

BTP is a transport protocol defined in the ETSI TS 103 836-5-1 V2.1.1 standard. It is used for Intelligent Transport Systems (ITS) applications.

BTP_A

Represents the BTP-A type message. BTP-A is used for interactive packet transport, where each packet is processed individually.

BTP_B

Represents the BTP-B type message. BTP-B is used for non-interactive packet transport, typically for destination-specific packets.
