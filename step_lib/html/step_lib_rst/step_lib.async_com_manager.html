step_lib.async_com_manager module

This module provides asynchronous MQTT communication functionality for ITS (Intelligent Transport Systems) messages.

The module implements a custom MQTT client that handles the publishing and receiving of ITS messages through a broker using the MQTT v5 protocol. It includes:

  • MqttQoS: Enumeration for MQTT Quality of Service levels

  • PublishTopicParams: Configuration dataclass for MQTT topic publishing parameters

  • AsyncComManagerParams: Configuration dataclass for the MQTT client settings

  • AsyncComManager: Main class extending aiomqtt.Client with ITS message handling capabilities

The implementation supports:

  • Asynchronous message publishing and subscription

  • Geohash-based topic organization

  • Message serialization and deserialization

  • Configurable QoS levels

  • TLS/SSL security

  • Comprehensive logging

  • Error handling for message processing

Example

params = AsyncComManagerParams.get_default() client = AsyncComManager(params) await client.connect() await client.publish_its_message(message=message, is_station_mobile=True)

async publish_its_message(message: FacilityMessage, address: GeoNetworkAddress, current_position: GpsLocation, is_station_mobile: bool) → None[source]

Publishes an ITS message to the appropriate topic based on the message type and geohash precision.

This method determines the publishing topic based on the message type, creates a geohash from the destination position, and publishes the serialized message to the resulting topic.

Parameters:

  • message (FacilityMessage) The ITS message to publish.

  • address (GeoNetworkAddress) The network address information.

  • current_position (GpsLocation) The current GPS position of the sender.

  • is_station_mobile (bool) Flag indicating whether the station is mobile.

Raises:

If deserialization is successful, the returned ItsMessage will contain the deserialized payload in its_payload. If an error occurs during deserialization, the error is logged and the message is returned with its original payload.

Returns:

A message container with topic, payload, QoS, retain flag, message ID,

properties, and the deserialized ITS payload if successful.

Return type:

ItsMessage

Raises:

StopAsyncIteration Implicitly when the underlying iterator is exhausted.

Note

This method catches all exceptions during deserialization to ensure the iterator continues functioning even if individual messages cannot be properly deserialized.

class step_lib.async_com_manager.MqttQoS(*values)[source]

Bases: IntEnum

MqttQoS: Enumeration class for MQTT Quality of Service levels.

The MQTT protocol provides three QoS levels for message delivery:

QOS_0

At most once delivery. The message is delivered according to the best efforts of the operating environment. Message loss can occur.

QOS_1

At least once delivery. The message is assured to arrive but duplicates may occur.

QOS_2

Exactly once delivery. The message is assured to arrive exactly once. This is the highest quality of service level and has the highest overhead.
